# v1.0.0

**Breaking Changes**

- renamed type `Widget` to `WidgetComponent` to resolve naming collision with the `Widget` component — update type annotations and imports accordingly
- removed generic from `DefaultNodeOptions` — replace `DefaultNodeOptions<{ myProp: boolean }>` with `DefaultNodeOptions & { myProp: boolean }` in custom widget option types
- renamed `Node` to `JsonNode` to avoid collision with the native `Node` type
- prefer `x-options` over `options` as schema keyword; changed format-ids to `x-widget`
- set draft-2020-12 as the default JSON Schema draft

**Features**

- add: draft agnostic support — editor now works across multiple JSON Schema drafts (draft-04, draft-07, draft-2019-09, draft-2020-12)
- add: drag and drop support between arrays
- add: drop validation for draggable items
- add: selection feature with `SelectionProvider`
- add: `WidgetComponent` type exported from `@sagold/react-json-editor`
- add: `doc-generator` promoted to standalone package `@sagold/doc-generator`
- add: `optional oneOfIndex` to value nodes
- add: package exports for all widgets and types
- fix: `WidgetProps.node` typed as `T` so options type is correctly inferred from the passed node
- upgrade: React 19, Mantine 8, Storybook/Vite, json-schema-library v10

## Upgrade to v1.0.0

**`Widget` type renamed to `WidgetComponent`**

```ts
// before
import { type Widget } from '@sagold/react-json-editor';
const myWidget: Widget<StringNode> = ...;

// after
import { type WidgetComponent } from '@sagold/react-json-editor';
const myWidget: WidgetComponent<StringNode> = ...;
```

Note: the `Widget` field on `WidgetPlugin` objects is unchanged — only the *type* was renamed.

**`DefaultNodeOptions` generic removed**

```ts
// before
export type MyOptions = DefaultNodeOptions<{
    myProp?: boolean;
}>;

// after
export type MyOptions = DefaultNodeOptions & {
    myProp?: boolean;
};
```

## Upgrade to v0.35.0

Previously `useEditor` was used by

```ts
const [node, editor] = useEditor(settings);
const WidgetComponent = editor.getWidget(node);
return <WidgetComponent editor={editor} node={node} />
```

as of `v0.35.0` you will change usage to

```ts
const editor = useEditor(settings);
if (editor == null) {
  return null;
}
const node = editor.getNode();
const WidgetComponent = editor.getWidget(node);
return <WidgetComponent editor={editor} node={node} />
```

for convinience you may use the `Widget` component simplifying the example to

```ts
import { Widget } from "react-json-editor";
const editor = useEditor(settings);
return <Widget editor={editor} />
```

**Note** `useEditorPlugin` works well with this change so you can keep the current usage, e.g.

```ts
import { Widget } from "react-json-editor";
const editor = useEditor(settings);
usePlugin(editor, myPlugin)
return <Widget editor={editor} />
```
